Book Notes: Nginx HTTP Server, 2nd Edition, Nedelcu, 2013

Clement Nedelcu, Packt Publishing, July 2013

Preface

  • Russian project, originally started in 2002. Now used for a number of very high traffic domains.
  • Uses asynchronous sockets, doesn't spawn a process per request. One process per core can handle thousands of connections.
  • Much simpler config and admin than Apache.
  • Open source, BSD license, has module system.

Chapter 1: Downloading and Installing Nginx

  • Install common prerequisites:
    aptitude install build-essentials libpcre3 libpcre3-dev zlib1g zlib1g-dev openssl openssl-dev
  • Download source.
  • Configure, make, make install. Make sure to check configure options for modules, SSL support, etc.
  • Test the config with the -t switch, then start it up.
  • Optionally set it up as a service with an init like the following, from http://wiki.nginx.org/Upstart: 
    # nginx
 
description "nginx http daemon"
author "George Shammas <georgyo@gmail.com>"
 
start on (filesystem and net-device-up IFACE=lo)
stop on runlevel [!2345]
 
env DAEMON=/usr/sbin/nginx
env PID=/var/run/nginx.pid
 
expect fork
respawn
respawn limit 10 5
#oom never
 
pre-start script
        $DAEMON -t
        if [ $? -ne 0 ]
                then exit $?
        fi
end script
 
exec $DAEMON

Configuration Directives

  • Base conf file is in [prefix]/conf/nginx.conf.
  • Lines starting with # are comments.
  • Lines in the format setting-name [whitespace] value[, value,...]; are directives.
