The Knapsack Problem

Input: A set of items S = {1, ..., n}, where item i has size si and value vi. A knapsack capacity is C.

Problem: Find the subset S' ⊂ S that maximizes the value of ∑i ∈ S' vi, given that ∑i ∈ S' si ≤ C; i.e., all the items fit in a knapsack of size C.

Variant 1: Subdivision Allowed: If you can subdivide the individual items, the best technique is a greedy algorithm that computes value per size, then fills from most to least until the knapsack is full.

Variant 2: 0/1 (No Subdivision):

  • Items are worth the same, but differ in size: Fill from smallest to largest.
  • Items are the same size, worth different amounts: Fill from cheapest to most expensive.
  • Items have the same price per unit: Attempt to minimize the empty space in the knapsack. That's an NP-Complete problem, but has good heuristics. Also called the "subset sum" problem. Can be addressed by "integer partitioning," wherein you partition S into two subsets A and B such that ∑a ∈ A a = ∑b ∈ B b, or attempt to make the difference as small as possible. That's a bin packing problem with two bins, or a knapsack problem with two half sized sacks—again, NP-complete.
  • All sizes are relatively small integers: This is good for C ≤ 1,000, but not for C ≥10,000,000.

Bin Packing    NP-complete

Input: A set of i items with sizes d1, …, dn. A set of m bins with capacity c1, …, cm.

Problem: Store all the items using the smallest number of bins.

Even the simplest bin packing problems are NP-complete, so you have to use heuristics instead of worst-case optimal algorithms. Thankfully, the usable heuristics are relatively simple. Factors to consider:

  • What are the shapes and sizes of the objects?
  • If the problem is one dimensional: it is a special case of the knapsack problem.
  • If all boxes are the same size and shape: Repeatedly filling each row gives a reasonable if not optimal solution.
  • Are there limitations on orientation and placement of items?
  • Is the problem online or offline? Knowing the full set of items to pack at start time is 'offline' computable, getting them one at a time is an 'online' version.

Offline heuristics:

  • Order by size or shape, then insert. Options for insertion:
    1. Select the first or leftmost bin an object fits in.
    2. Select the bin with the most room.
    3. Select the bin that provides the tightest fit.
    4. Select a random bin.
  • "First-fit decreasing" appears to be the best heuristic:
    1. Sort objects in decreasing size order.
    2. Insert each object one by one into the first bin that fits.
    3. If no bin fits, start a new bin.
    You essentially pack the big stuff first, then fill in the cracks. It usually works in O(n log n + bn) time, where b ≤ min(n,m) is the number of bins actually used. You can get it to O(n log n) if you use a binary tree to keep track of the space remaining in each bin.

Set Cover    NP-complete

Input: A collection of subsets S = {S1, …, Sm} of the universal set U = {1, …, n}.

Problem: What is the smallest subset T of S whose union equals the universal set—i.e., ⋃|T|i=1 Ti = U?

The set cover problem arises when you try to efficiently acquire items packaged in a fixed set of lots. You want a collection of at least one of each distinct type of item while buying as few lots as possible.

Variations on set cover:

  • Can you cover an element more than once? If you have the freedom to cover more than once, use it—it's usually more efficient.
  • Are the sets derived from the edges or vertices of a graph? The vertex cover problem is a subset of set covering.
  • Do the subsets contain only one element each? This case can be solved efficiently because it becomes finding a maximum matching in a graph. Once you go to sets of 3 it becomes NP-complete.
  • Do you want to cover elements with sets, or sets with elements?

Greedy is the best heuristic for set cover:

  • Pick the largest subset, remove its elements from U.
  • Repeatedly add the subset containing the largest number of remaining elements, until all are covered.

If you use a linked list and a bounded height priority queue, you can get the heuristic down to something like O(S) time, where S is the union of the sizes of all the subsets.
