Developed starting in the late 1980's by Guido van Rossum. The 2.0 release in 2000 brought in automated garbage collection and unicode support, and the 3.0 release in 2008 added features and refinements, many of which were also backported to 2.6 and 2.7. Python has a rather specific design philosophy, typified by statements like, Explicit is better than implicit. and Simple is better than complex. See PEP 20 for the original "Zen of Python" document.

Python is in many ways diametrically opposed to perl. Where perl was created by a linguist, python is the creation of a mathematician. Perl has "more than one way to do it," while python generally strives to have a single, canonical way of expressing specific operations. Some of the more striking features of python are:

  • White space is meaningful and proper formatting is enforced by the interpreter—consequently python programs tend to be very readable (though not friendly to careless copy-paste jobs).
  • Set operations are built into the language, so performing unions and intersections, as well as set comparisons, is syntactically simple.
  • Single statement, anonymous functions (lambdas) can act as function literals, allowing a (limited) form of functional programming.
  • List and dictionary 'comprehensions' allow quick filtering / mapping of composite variables.

Truth Values

          
Note that in Python, all objects have a Boolean value, and may be used in a test statement. Additionally, Python's Boolean operators will stop evaluating a statement as soon as a result can be determined. This is similar to the convention in some other languages of having a strong and weak set of logical operators, with the strong operator having higher precedence and forcing full evaluation of a statement, and the weak operator returning as soon as a value is found.

          
There are several notable knock-on effects of Python's short-circuit logical evaluations:

          
    
            
  • You can use or statements to select the first non-empty object from a list of objects: item = X or Y or Z or None
    • 
            
  • You should be intentional about using or to evaluate the return values of functions, since in a statement like f1() or f2(), f2() will not run when f1() returns True.
    • 
          

          
Additionally, it is important to note that the values True and False are internally the same as the integers 1 and 0 respectively. Thus, initializing a variable to False is the same as setting it to 0.

Comments

          
Python has two comment styles: inline comments prefaced with #, and 'docstrings,' which are multi-line comments encosed in triple quotes (""" ... """). Docstrings are parsed by various documentation tools to present interactive documentation for your code, while inline comments are typically used for one-off notes only meant to be seen by someone actually reading your code directly.


# This is an inline comment.

""" 
This is a docstring. 
"""

A Note on Operator Symbols vs. Named Operations in Python

            
Python has a full complement of operator symbols: +, -, *, /, %, **, //, and so on. In common usage they will appear as in most other languages. However, it is worth understanding the mechanism by which those operators are applied to Python objects, since they represent an abstraction that can be overloaded or repurposed.

            
Since everything in Python is an object, applying binary or unary operations to operands actually means applying them to objects. That being the case, each object must define how it will respond to built in operators—when you combine two numbers with +, you are actually asking two Number objects to take part in a summation, rather than combining literal numeric primitives.

            
Python accomplishes this through a series of operator overloading methods, which the common operator symbols are merely proxies for. So, for instance, adding two numeric objects with the + operator actually means calling the __add__ method behind the scenes.

            
These operator overloading methods are inherited by any object type capable of responding to their corresponding operator symbol, and you can overload the methods to define how objects of your own classes will respond to built in operators.

A note on 'empty' blocks in Python:

        
Due to Python's whitespace-delimited syntax the language requires an explicit no-operation statement: pass. Without pass, creating placeholder code that included compound statements would be difficult. For example:


#TODO: flesh out the push and pop methods:
def push():
  pass

def pop():
  pass

        
Leaving a completely empty function body would cause a syntax error, since the def statement opens a block. Without the pass statement, developers would be forced to include some other no-operation statement, like True or 1—pass makes clear that the line is intended to be a no-op.

        

      

      

        
Blocks

        
Python blocks are delimited by preceeding whitespace. The amount of whitespace denoting a block indentation is arbitrary between programs, but within a program must be internally consistent. You can use four spaces, two spaces, literal tabs, etc., but once you establish an indentation method in a file (on first usage), you must continue to use that for the remainder of the file.

        
Python does not have "naked" blocks

Additional Python Iterators: Comprehensions

              
Python can loop and iterate using while and for, but also provides 'comprehensions' as an additional iteration structure. A comprehension is a statement that returns a list (or dictionary, or set) of items which have been filtered and/or transformed. Comprehensions take the general form:


[ expression for item in iterable if condition ]

              
More specifically:


[
  item * 2                  # Mapping statement for each item through the filter
  for                       # Keyword, multiples allowed
  item                      # Declaration of loop variable for mapping statement
  in                        # Keyword
  myList                    # Some iterable
  if item > 5               # Filtering condition (optional)
]

              
Concrete examples of list comprehensions:


one2nine = [n for n in range(1,10)]               # Returns [1,2,3,4,5,6,7,8,9]
evens    = [n for n in range(1,10) if n % 2 == 0] # Returns [2,4,6,8]
doubles  = [n*2 for n in range(1,10)]             # Returns [2,4,6,8,10,12,14,16,18]
tuples   = [(n,n*2) for n in range(1,5)]          # Returns [(1,2),(2,4),(3,6),(4,8)]

              
It is valid to use more than one for statement in a comprehension (each of which may have an if statement), so you can create comprehensions equivalent to nested loops:


rows = range(1,4)
cols = range(1,4)

# Will return [(1, 1), (1, 2), ..., (3, 2), (3, 3)]
table_cells    = [(r, c) for r in rows for c in cols]

# Will return [(1, 1), (2, 2), (3, 3)]
identity_cells = [(r, c) for r in rows for c in cols if r == c]

# Will return [(3,3)]
ident_over_2   = [(r, c) for r in rows if r > 2 for c in cols if r == c]


              
Dictionary comprehensions:


# Will return {97: 'a', 98: 'b', 99: 'c', 100: 'd', 101: 'e'}
charcodes = { number : chr(number) for number in range(97,102) }


              
Set comprehensions:


numbers = [1,2,3,1,1,5,9,5]

unique_odds = { n for n in numbers if n % 2 }   # Returns set([1, 3, 5, 9])


              
Note: Python does not have comprehensions for tuples. If you see syntax like the following, it is a 'generator' expression, not a tuple comprehension. Generators are expressions that hold their state from call to call, and are capable of acting like iterables without actually holding all their possible values in memory--because those values are generated on each call.



my_generator = (n for n in range(1,100))  # Creates a generator object

next(my_generator) # returns 1
next(my_generator) # returns 2


              
Each iteration of a generator function will produce the next value in the sequence, but at no point is it necessary for my_generator to actually contain a ninety-nine item list of numbers.

            

          

        

      

      

        
Functions

        
Python functions serve the same dual purpose as in many object-oriented languages: promoting code modularity and reuse, and serving as executable members of instantiated objects. In practical terms functions are defined using the def or lambda statements, called by name with parenthetical arguments, and return values using the return and yield statements.

        
Python functions (both those produced by def and lambda) are first class citizens, and taking a function reference is done as with any other variable:


def myFunc(myArg):
  return myArg

myRef = myFunc
myRef('an argument')    # Returns the string 'an argument'


        

          
Declaration

          
Python functions, like other named entities, are not declared separately from their definition.
