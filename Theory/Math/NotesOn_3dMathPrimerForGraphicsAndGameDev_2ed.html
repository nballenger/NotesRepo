1.1 1D Mathematics

  • Number line, real numbers, etc.

1.2 2D Cartesian Space

  • Origin, x/y coordinates

1.3 3D Cartesian Space

  • x/y/z coordinates; xy, xz, yz planes
  • Left handed and right handed 3D coordinate spaces
    • If two systems have the same handedness, they can be rotated to match
    • Left handed space:
      • Index finger points up at +y
      • Middle finger points forward at +z
      • Thumb points right at +x
    • Right handed space, thumb points left at +x
    • In a left handed space, the direction of rotation about an axis of rotation is determined by making a closed fist, thumb pointing up at the + end of the axis of rotation. The rotation direction is in the same direction as the fingers curl.
    • To convert handedness, swap the positive and negative ends of a single axis, or exchange two axes.
    • This book uses left handed coordinate system
      • +x is right
      • +y is up
      • +z is forward
  • Summation notation using Σ
  • Product notation with Π
  • Closed and open intervals with [], ()
    • closed interval includes the endpoints, []
    • open interval excludes the endpoints, ()
  • Degrees and radians
  • Radian is arc length on a unit circle, meaning 2πrads is 1 circumfrence
  • The x,y endpoint of a ray rotated about the origin can be described as cos Θ = x, sin Θ = y, where Θ is the angle of rotation.
  • The trig functions, defined in regards to a unit circle:
    • cos Θ = x
    • sin Θ = y
    • sec Θ = 1 / cos Θ
    • tan Θ = sin Θ / cos Θ
    • csc Θ = 1 / sin Θ
    • cot Θ = 1 / tan Θ = cos Θ / sin Θ
  • The trig functions can also be defined in relation to hyp, opp, adj sides of a triangle, where the adj side is on the x axis, the opp follows the y axis, and the hyp connects the origin to the end of the opp side.
    • cos Θ = adj / hyp
    • sin Θ = opp / hyp
    • tan Θ = opp / adj
    • sec Θ = hyp / adj
    • csc Θ = hyp / opp
    • cot Θ = adj / opp
  • Identities based on the symmetry of the unit circle
    • sin (- Θ) = - sin Θ
    • cos (- Θ) = cos Θ
    • tan (- Θ) = - tan Θ
    • sin (π/2 - Θ) = cos Θ
    • cos (π/2 - Θ) = sin Θ
    • tan (π/2 - Θ) = cot Θ

Chapter 2: Vectors

2.1 Mathematical Definition of Vector, and Other Boring Stuff

  • Vectors are arrays of numbers
  • The dimension of a vector is how many numbers it contains
  • Vectors may be of any positive dimension
  • Vectors may be written horizontally or vertically, in brackets
  • Book conventions for different types of variable:
    • Scalar variables are lowercase, italic roman or greek letters
    • Vector variables are lowercase, boldface letters
    • Matrix variables are uppercase, boldface letters

2.2 Geometric Definition of Vector

  • A vector is a directed line segment with magnitude and direction
  • Magnitude is the length of the vector
  • Direction is the way it points in space
  • Typically graphically represented as an arrow
  • Vectors do not have a position, because they represent things like displacement or velocity (I am traveling at 50mph northeast.)

2.3 Specifying Vectors with Cartesian Coordinates

  • When using cartesian coordinates to describe a vector, each coordinate indicates a signed displacement off an axis.
  • For example, vector [x,y] is displaced x units along the x-axis, and y units along the y-axis.

2.3.1 Vector as a sequence of displacements

  • A vector is a combination of all its displacements.

2.3.2 The Zero Vector

  • For any given vector dimension there is a special vector (the zero vector) with zeroes in every position. 3d zero vector is [0,0,0], for instance.
  • Denoted by boldface 0 character, for any dimension.
  • Think of it as a way to communicate "no displacement."
  • The zero vector of a given dimension is the additive identify for the set of vectors in that dimension.

2.4 Vectors versus Points

  • Vectors describe displacements, and therefore can describe relative position--a point displaced from some agreed upon origin.
  • Any point can be described as a vector from the origin.

2.5 Negating a Vector

  • Every vector v has an additive inverse -v of the same dimension as v such that v + (-v) = 0.

2.5.1 Official Linear Algebra Rules

  • To negate a vector of any dimension, simply negate each component of the vector: -[a1, a2, ..., an] = [-a1, -a2, ..., -an]

2.5.2 Geometric Interpretation

  • Same magnitude, opposite direction.

2.6 Vector Multiplication by a Scalar

Official linear algebra rules

  • Result of multiplying a vector and a scalar is a parallel vector, with a different length and possibly an opposite direction.
  • The process is to multiply each element of the vector by the scalar.
  • Dividing by a scalar is equivalent to multiplication by the reciprocal of the scalar.
  • Scalar/vector multiplication and division happen BEFORE addition and subtraction in OOP
  • A scalar may not be divided by a vector.
  • A vector may not be divided by another vector.
  • Vector negation can be viewed as multiplying by -1

Chapter 4: Introduction to Matrices

4.1 Mathematical Definition of Matrix

4.1.1 Matrix Dimensions and Notation

  • Matrix of r rows and c columns is called an r by c matrix.
  • Notation is a grid in full height brackets.
  • In the book matrices are uppercase bold, element access is subscript notation of the type mij
    • , where i is the row and j is the column.
  • A matrix in straight vertical lines is the determinant of a matrix.

4.1.2 Square Matrices

  • Diagonal elements of a square matrix are those for which i == j. In aggregate they form the 'diagonal' of the matrix.
  • If all non-diagonal elements are zero, the matrix is a 'diagonal matrix.'
  • A diagonal matrix with all diagonal elements set to 1 is an 'identity matrix.'
  • Multiplication of a matrix by an identity matrix produces the original matrix.

4.1.3 Vectors as Matrices

  • A 1 x n matrix is a 'row vector'.
  • An n by 1 matrix is a 'column vector'.
  • It is very important when dealing with matrices to be clear on whether you are using a row vector or column vector.

4.1.4 Matrix Transposition

  • Given an r x c matrix M, the transpose (MT) is the c x r matrix where the columns are formed by the rows of M.
  • MTij == Mji
  • Transposition of a row vector produces a column vector, and vice versa.
  • Transposition of a transpose matrix produces the original matrix.
  • Any diagonal matrix is equal to its transpose, including the identity matrix.

4.1.5 Multiplying a Matrix with a Scalar

  • Multiplication of a matrix M by a scalar k results in a matrix of the same dimension as M, where each element of kM is the product kMij.

4.1.6 Multiplying Two Matrices

  • Only some matrix products are defined. An r x n matrix A may be multiplied by an n x c matrix B.
  • AB is an r x c matrix.
  • Example: A is 4 x 2, B is 2 x 5. AB is a 4 x 5 matrix.
  • The elements of AB are computed as follows, for product matrix C: cij = Σ aikbkj, for k 1..n
  • Written a different way, each element cij is the dot product of row i in A and column j in B.
  • Notes of interest about matrix multiplication:
    • Multiplying any matrix M by a square matrix S results in a matrix of the same size as M, if MS is defined.
    • Matrix multiplication is non-commutative. AB != BA.
    • Matrix multiplication is associative. (AB)C == A(BC)
    • Matrix multiplication is associative in multiplication by a scalar or vector. (kA)B = k(AB) = A(kB), and (vA)B = v(AB)
    • Transposing the product of two matrices is the same as the product of their transpose in reverse order. (AB)T = BTAT

4.1.7 Multiplying a Vector and a Matrix

  • Since a vector can be a row or column matrix, you can multiply it by another matrix if the product is defined.
  • In 3 dimensions, a 3 element vector may be multiplied by a 3 x 3 square matrix.
  • The defined products for a 3 x 3 M, a row vector r, and a column vector c are rM = row vector and Mc = column vector.
  • Case rM: Take the row vector r and the three column vectors that comprise M. The three terms of the output vector that is the product of rM are each sum of the products of x times the first element, y times the second, and z times the third.
  • Case Mc: Take the column vector c and the three row vectors that make up M. The three terms of the output vector that is the product of Mc are each the sum of the products of x time the first element, y times the second element, and z times the third element of the first, second, and third row vectors.
  • Vector times matrix multiplication distributes over vector addition. (v + w)M == vM + wM
  • The result of the multiplication is a linear combination of the rows or columns of the matrix.

4.1.8 Row versus Column Vectors

  • rM and Mc both produce three element vectors, but they do not produce the same vector elements, so it is very important to know which of rM or Mc you are performing.
  • Because the output of the operations is in each case a vector, you can concatenate multiplication operations with multiplie matrices. For a row vector that reads left to right, as in rABC, whereas a column vector will concatenate right to left: CBAc.
  • Row vectors lead to very wide equations, so are out of favor in mathematics--column vectors make for more readable equations.
  • For most video game programming, readable code is more important, so row vectors are a more typical choice.
  • Different books use different conventions, so be careful. Same goes for APIs.
  • Well encapsulated code prevents the need to access individual matrix elements, which prevents mistakes.

4.2 Geometric Interpretation of Matrix

  • A square matrix can describe most or all linear transformations.
  • A linear translation preserves straight and parallel lines, and the origin does not move (there is no translation).
  • Other geometric properties like length, angle, area, volume, are possibly altered by the transformation.
  • A linear transformation may stretch the coordinate space, but does not curve or warp it.
  • The transformations this includes:
    • rotation
    • scale
    • orthographic projection
    • reflection
    • shearing
  • It is important to be able to visualize the transformation that a matrix represents, and vice versa.
  • Example is given of multiplying the three standard basis vectors by an arbitrary matrix M: i M = 1 0 0 m11 m12 m13 m21 m22 m23 m31 m32 m33 = m11 m12 m13 j M = 0 1 0 m11 m12 m13 m21 m22 m23 m31 m32 m33 = m21 m22 m23 k M = 0 0 1 m11 m12 m13 m21 m22 m23 m31 m32 m33 = m31 m32 m33
  • The first row of M contains the result of the transformation on i, the second row on j, and the third row on k.
  • Any vector can be written as a linear combination of the standard basis, v = vxi + vyj + vzk
  • "The result of a vector x matrix multiplication is a linear combination of the rows of the matrix. The key is to interpret those row vectors as basis vectors. In this interpretation, matrix multiplication is simply a compact way to encode the operations for coordinate space transformations.
  • Substituting p, q, and r as names for our basis vectors, we can write vector v times matrix M as: v M = vx vy vz p q r = vxp + vyq + vzr
  • If you understand how a matrix transforms the standard basis vectors, you understand the transformation it represents. Since the results of transforming the standard basis are the rows of the matrix, you interpret those rows as the basis vectors of a coordinate space.
  • Given a desired transformation, you can derive a matrix that represents it by figuring out what the transformation does to the basis vectors, and then put those transformed vectors into the matrix as rows.
  • "Once we understand that coordinates are best understood as coefficients in a linear combination of the basis vectors, we really know all the math we need to know to do transformations.
  • Core concepts of this section:
    • The rows of a square matrix can be interpreted as the basis vectors of a coordinate space.
    • To transform a vector from the original coordinate space to the new coordinate space, we multiply the vector by the matrix.
    • The transformation from the original coordinate space to the coordinate space defined by these basis vectors is a linear transformation. A linear transformation preserves straight lines, and parallel lines remain parallel. However, angles, lengths, areas, and volumes may be altered.
    • Multiplying the zero vector by any square matrix results in the zero vector. Therefore, the linear transformation represented by a square matrix has the same origin as the original coordinate space--the transformation does not contain translation.
    • We can visualize a matrix by visualizing the basis vectors of the coordinate space after transformation. These basis vectors form an 'L' in 2D, and a tripod in 3D. Using a box or auxiliary object also helps in visualization.

4.3 The Bigger Picture of Linear Algebra

  • Linear algebra was invented to manipulate and solve systems of linear equations.
  • Matrix notation was invented to avoid having to write out all the symbols in a system of equations.
  • Examples of large systems of equations that must be solved are within the physics engine of a 3D game, or in least squares approximation and data-fitting applications.
  • Many problems which have traditionally been neither discrete or linear are being approximated through systems of linear equations given the amount of computing power we have now.
  • The challenge is most often figuring out how to transform a problem into a matrix.
  • This book is intended to bridge a gap between linear algebra texts and game programming texts, but there are tons of advanced areas we're not touching on.
  • Great resource is Prof. Gilbert Strang's lectures from MIT OpenCourseWare. He also writes textbooks aimed at engineers, though they may require the lectures to really understand.

5.1 Rotation

5.1.1 Rotation in 2D

  • 2D rotation is rotation about a point. Since this is about linear transformations, this means rotation about the origin.
  • A 2D rotation about the origin has one parameter, Θ, which defines the angle rotated by. Standard convention is counterclockwise rotation for positive Θ, clockwise rotation for negative.
  • In rotation about the origin, the basis vectors of your transformation matrix can be derived in relation to Θ: R Θ = p' q' = cosΘ sinΘ -sinΘ cosΘ

5.1.2 3D Rotation about Cardinal Axes

  • In 3D rotation occurs around an axis rather than a point.
  • An axis of rotation does not have to be one of the cardinal axes, though this section only looks at those.
  • There's no translation, so this is all about rotation about an axis passing through the origin.
  • For rotation you need to know which direction is positive and which is negative--use the left hand rule to find this.
  • Rotation about the x axis: R Θ = p' q' r' = 1 0 0 0 cosΘ sinΘ 0 -sinΘ cosΘ
  • Rotation about the y axis: R Θ = p' q' r' = cosΘ 0 -sinΘ 0 1 0 sinΘ 0 cosΘ
  • Rotation about the z axis: R Θ = p' q' r' = cosΘ sinΘ 0 -sinΘ cosΘ 0 0 0 1

5.1.3 3D Rotation about an Arbitrary Axis

  • If an axis passes through the origin, we can rotate around it without translating.
  • Again, Θ is the amount of rotation about the axis, and the axis of rotation is defined by the unit vector n^
  • Our goal is, for a vector v to produce a matrix R such that the result v' is equivalent to rotating v about n^ by the angle Θ: v' = v R n ^ Θ
